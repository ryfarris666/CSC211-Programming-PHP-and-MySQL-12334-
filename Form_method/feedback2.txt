<h3>Form Method Explation<h3>
<p><strong>Method="post" means:</strong> Form data is sent inside the HTTP request body, 
making it invisible in the URL. This approach has no size limitations for data, 
prevents bookmarking form submissions, and provides better security for sensitive 
information like passwords and personal details.</p>

<p><strong>Contrast with method = “get”:</strong>With GET, data is appended to the URL as name/value pairs, 
making it visible in the browser. It’s limited to 3000 characters, allows users to 
bookmark results, but shouldn’t be used for sensitive data like passwords due to its 
visibility.</p>
